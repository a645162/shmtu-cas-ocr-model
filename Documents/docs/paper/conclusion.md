# 6. 结论

## 6.1 总结

本文针对上海海事大学 CAS 统一认证平台的算术验证码识别问题，提出了基于 TriSlot Decoder 的端到端卷积神经网络模型。主要研究成果如下：

1. **TriSlot Decoder 架构**：利用可学习的槽位查询和多头注意力机制，在单次前向传播中实现验证码三字符的端到端识别，消除了传统方法中图像切割带来的误差累积问题。

2. **结构化损失函数**：设计了包含槽位顺序约束、重叠惩罚、右边界约束和注意力方差约束的损失函数体系，有效引导模型学习验证码的空间排列先验，相比纯分类损失提升约 0.8 个百分点。

3. **领域特定数据增强**：提出了二值化参数扰动策略，模拟不同预处理参数对验证码图像的影响，相比基础增强进一步提升约 0.4 个百分点。

4. **轻量化部署**：以 MobileNetV3-Small 为 backbone 的模型仅约 8M 参数，在验证集上达到 98%+ 的整体表达式准确率，导出为 NCNN fp16 格式后模型仅约 4 MB，CPU 推理延迟约 5ms，满足实时部署需求。

## 6.2 V1 与 V2 的演进

| 维度 | V1 | V2 |
|---|---|---|
| 架构理念 | 分而治之 (切割 + 独立分类) | 端到端 (共享特征 + 槽位解码) |
| 空间建模 | 无 (切割后丢失) | 槽位注意力隐式建模 |
| 模型管理 | 3 个独立模型 | 1 个统一模型 |
| 推理效率 | 3 次前向 | 1 次前向 |
| 部署体积 | ~100 MB | ~4 MB (NCNN fp16) |

V2 在所有维度上均优于 V1，验证了端到端方法在该任务上的优越性。

## 6.3 局限性与未来工作

本文工作存在以下局限性，值得进一步探索：

1. **数据规模**：当前数据集规模约 5000 条，更大规模的数据集可能进一步提升模型精度和泛化能力。

2. **验证码变体**：当前模型仅针对上海海事大学 CAS 验证码设计，对于其他风格（字符数量变化、噪声更强、扭曲更大）的验证码，需要调整模型结构。

3. **算术语义约束**：当前模型仅做分类，未利用算术表达式的语义约束（如 $9-2=7$ 中 result 必须正确）。引入算术约束可能进一步提升整体识别一致性。

4. **模型架构探索**：本文仅在 CNN backbone 范围内进行对比，未来可探索 Vision Transformer (ViT)、ConvNeXt 等现代架构。

5. **对抗鲁棒性**：本文未研究模型对对抗攻击的鲁棒性。验证码安全性评估需要考虑对抗样本的威胁模型。
