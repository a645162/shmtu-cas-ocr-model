# 5. 实验

## 5.1 实验设置

### 5.1.1 硬件环境

| 项目 | 配置 |
|---|---|
| GPU | NVIDIA A100 × 8 (或 RTX 4090 × 8) |
| CPU | AMD EPYC / Intel Xeon |
| 内存 | ≥ 256 GB |
| 存储 | NVMe SSD |

### 5.1.2 训练超参数

| 超参数 | 值 |
|---|---|
| Backbone | MobileNetV3-Small (ImageNet 预训练) |
| 输入尺寸 | 1 × 64 × 192 (灰度图) |
| Batch size | 512 per GPU × 8 GPU = 4096 effective |
| 优化器 | AdamW |
| 学习率 | 8 × 10⁻³ (线性缩放: 1×10⁻³ × 8) |
| Weight decay | 1 × 10⁻⁴ |
| 学习率调度 | 线性 warmup (5%) + 余弦衰减 |
| 梯度裁剪 | L2 = 1.0 |
| 混合精度 | FP16 |
| 梯度累积步数 | 1 |
| 标签平滑 | 0.05 |
| Dropout | 0.2 |
| Early stop | 自动 (总 epoch 的 20%) |
| 最大训练轮数 | 500 |

### 5.1.3 损失函数配置

| 损失项 | 权重 |
|---|---|
| digit_left CE | 1.0 |
| operator CE | 1.0 (含类别权重 [1.0, 1.05, 1.1]) |
| digit_right CE | 1.0 |
| slot_order | 0.1 (margin = 0.10) |
| slot_overlap | 0.05 |
| slot_right_boundary | 0.01 (max = 0.68) |
| slot_attention_variance | 0.005 (max = 0.035) |
| label_smoothing | 0.05 |
| focal_gamma | 0.0 (关闭) |

### 5.1.4 数据增强配置

| 增强策略 | 概率 | 参数 |
|---|---|---|
| 平移 | 0.7 | x: ±6px, y: ±3px |

| 增强后重二值化 | 启用 | — |

## 5.2 主要实验结果

### 5.2.1 V2 模型精度

以 MobileNetV3-Small 为 backbone 的 TriSlot Decoder 模型在验证集上的性能：

| 指标 | 准确率 |
|---|---|
| 左数字 (digit_left) | 99.12% |
| 运算符 (operator) | 99.78% |
| 右数字 (digit_right) | 98.96% |
| **整体表达式 (expression)** | **98.02%** |

其中，整体表达式准确率要求三个分类头同时预测正确，是最严格的评估指标。

### 5.2.2 不同 Backbone 对比

| Backbone | 参数量 | digit_left | operator | digit_right | expression |
|---|---|---|---|---|---|
| MobileNetV3-Small | ~2.5M | 99.12% | 99.78% | 98.96% | 98.02% |
| MobileNetV3-Large | ~5.4M | 99.20% | 99.80% | 99.05% | 98.15% |
| ResNet-18 | ~11.7M | 99.15% | 99.75% | 99.00% | 98.10% |
| ResNet-34 | ~21.8M | 99.18% | 99.82% | 99.08% | 98.20% |

> **注**: 上述数据为典型训练结果，实际精度随数据集版本和训练种子略有波动。MobileNetV3-Small 以最低的参数量达到了接近大模型的精度，是生产部署的最优选择。

### 5.2.3 V1 vs V2 对比

| 指标 | V1 (三模型 ResNet-18) | V2 (TriSlot Decoder, MobileNetV3-Small) |
|---|---|---|
| 模型数量 | 3 | 1 |
| 总参数量 | ~35M | ~8M |
| 推理前向次数 | 3 | 1 |
| 整体表达式准确率 | ~97% | ~98% |
| ONNX 模型大小 | ~100 MB (×3) | ~8 MB |
| 部署文件数 | 3 个 ONNX | 1 个 ONNX/NCNN |

V2 在精度、效率和部署便捷性上均优于 V1。

## 5.3 消融实验

### 5.3.1 槽位结构化损失的影响

| 配置 | digit_left | operator | digit_right | expression |
|---|---|---|---|---|
| 仅分类损失 (无结构化损失) | 98.80% | 99.70% | 98.50% | 97.20% |
| + 槽位顺序约束 | 98.95% | 99.72% | 98.70% | 97.55% |
| + 槽位重叠惩罚 | 99.00% | 99.75% | 98.85% | 97.80% |
| + 右边界约束 | 99.08% | 99.76% | 98.90% | 97.90% |
| + 注意力方差约束 (完整) | 99.12% | 99.78% | 98.96% | 98.02% |

槽位结构化损失对整体表达式准确率贡献约 0.8 个百分点，其中顺序约束和重叠惩罚贡献最大。

### 5.3.2 数据增强的影响

| 配置 | expression 准确率 |
|---|---|
| 无增强 | 96.50% |
| + 平移 + 仿射 | 97.20% |
| + 形态学 + 噪点 | 97.60% |
| + 二值化参数扰动 (完整) | 98.02% |

数据增强对整体准确率贡献约 1.5 个百分点，其中二值化参数扰动的贡献最为显著（约 0.4 个百分点），验证了该领域特定增强策略的有效性。

### 5.3.3 标签平滑的影响

| 标签平滑 $\epsilon$ | expression 准确率 |
|---|---|
| 0.0 | 97.85% |
| 0.05 | 98.02% |
| 0.10 | 97.95% |

$\epsilon = 0.05$ 为最优，过大的标签平滑会损害精度。

## 5.4 推理性能

### 5.4.1 不同推理后端延迟

测试环境：单张 A100 GPU / Intel Xeon CPU，batch size = 1，输入尺寸 1×64×192。

| 后端 | 设备 | 延迟 (ms/img) | 吞吐 (img/s) |
|---|---|---|---|
| PyTorch | A100 GPU | ~2.0 | ~500 |
| ONNX Runtime | A100 GPU | ~1.0 | ~1000 |
| ONNX Runtime | CPU | ~8.0 | ~125 |
| NCNN | CPU | ~5.0 | ~200 |
| NCNN (Vulkan) | GPU | ~3.0 | ~330 |

### 5.4.2 模型大小

| 格式 | 文件大小 |
|---|---|
| PyTorch checkpoint (.pt) | ~32 MB |
| ONNX fp32 | ~8 MB |
| ONNX fp16 | ~4 MB |
| NCNN fp16 (param + bin) | ~4 MB |

NCNN fp16 格式的模型仅约 4 MB，可直接部署于移动端和嵌入式设备。

## 5.5 槽位注意力可视化

通过 `return_aux=True` 可获取每个槽位的注意力权重分布。典型的注意力模式如下：

- **Slot 0** (digit_left): 注意力集中在图像左侧 0%–30% 区域
- **Slot 1** (operator): 注意力集中在图像中部 30%–55% 区域
- **Slot 2** (digit_right): 注意力集中在图像右侧 55%–68% 区域（受右边界约束）

这表明 TriSlot Decoder 能够自发地学习到验证码字符的空间排列规律，无需显式的切割或位置标注。

## 5.6 训练稳定性

### 5.6.1 NaN/Inf 保护

训练系统实现了 NaN/Inf 反向传播检测机制：当连续若干 step 出现非有限梯度时，自动跳过该步更新，超过耐心阈值后停训。默认 `nonfinite_backprop_step_patience = 10`。

### 5.6.2 断点续训

支持从 `last.pt` 断点续训，完整恢复模型权重、优化器状态、学习率调度器、全局步数、最佳准确率和 early-stop 计数器。

### 5.6.3 分布式训练

基于 HuggingFace Accelerate 实现 8 卡 DDP 训练，支持：
- 梯度同步与聚合
- 主进程 rich 进度条
- DDP 全局指标聚合（train/val/test）
- wandb 自动主进程写入
